# Contributing to Django Toastify

Thank you for your interest in contributing to Django Toastify! We welcome contributions from the community and appreciate your effort to help us improve this project.

## Code of Conduct

Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md) to help us maintain a welcoming and inclusive community.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion for improvement:

1. Check the [existing issues](https://github.com/prakash-thapa/django-toastify/issues) to avoid duplicates
2. Open a new issue with a clear title and description
3. Include steps to reproduce (for bugs) or use cases (for features)
4. Provide any relevant code snippets or error messages

### Submitting Pull Requests

1. **Fork the repository** and clone it locally
   ```bash
   git clone https://github.com/your-username/django-toastify.git
   cd django-toastify
   ```

2. **Create a virtual environment** and install dependencies
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # On Windows: .venv\Scripts\activate
   pip install -e .
   ```

3. **Create a new branch** for your feature or fix
   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Make your changes** following the project's code style
   - Keep commits focused and atomic
   - Write clear, descriptive commit messages
   - Test your changes thoroughly

5. **Push your branch** and open a Pull Request
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Describe your changes** in the PR description
   - Link any related issues
   - Explain the motivation and implementation
   - Highlight any breaking changes

### Development Guidelines

- Follow PEP 8 style guidelines for Python code
- Keep JavaScript code clean and readable
- Document your code with comments where necessary
- Update README.md if you're adding new features or changing behavior
- Add tests for new functionality

### Areas for Contribution

- Bug fixes and issue resolution
- Documentation improvements
- Feature requests and enhancements
- Theme and CSS improvements
- Performance optimizations
- Accessibility improvements
- Browser compatibility enhancements

## Questions?

Feel free to open an issue or discussion if you have questions about contributing. We're here to help!
